# Gitalyze

A GitHub analytics dashboard to visualize user/repository stats, compute a profile score breakdown, and optionally generate improvement recommendations.

## Features

- User profile analytics + score breakdown
- Repository detail analytics
- Optional AI recommendations (Groq) with caching + deterministic fallback
- SEO-friendly metadata with canonical URLs, plus `/sitemap.xml` and `/robots.txt`

## Tech stack

- Next.js 15 (App Router)
- React 19
- TypeScript
- Tailwind CSS
- Bun (recommended) or Node.js

## Quickstart (Local)

1. Install dependencies:
   - `bun install`
2. Create your local env file:
   - `cp .env.example .env.local`
3. Start the dev server:
   - `bun run dev`

Production build locally:

- `bun run build`
- `bun run start`

## Environment variables

Environment variables are documented in `.env.example`. The essentials:

| Name | Required | Purpose |
| --- | --- | --- |
| `SITE_URL` | Yes | Server-side canonical host for metadata, sitemap, and robots |
| `NEXT_PUBLIC_SITE_URL` | Yes | Client-side canonical host |
| `GITHUB_API_TOKEN` | No | Higher GitHub API rate limits |
| `GROQ_API_KEY` | No | Enable AI recommendations |
| `GROQ_MODEL` | No | Groq model selection |
| `GROQ_TIMEOUT_MS` | No | Groq request timeout |
| `GROQ_MAX_RETRIES` | No | Groq retry count |
| `RECOMMENDATION_CACHE_TTL_MS` | No | Server cache TTL for AI results |
| `GROQ_COOLDOWN_MS` | No | Cooldown after rate limiting |
| `NEXT_PUBLIC_RECOMMENDATION_CACHE_TTL_MS` | No | Client cache TTL |
| `FORCE_AI_RECS` | No | Force deterministic (non-AI) recommendations |

## Scripts

- `bun run dev` — start dev server
- `bun run lint` — TypeScript typecheck (`tsc --noEmit`)
- `bun run build` — production build
- `bun run start` — start production server

## SEO & Sitemap

- Set `SITE_URL` and `NEXT_PUBLIC_SITE_URL` so canonical URLs and Open Graph links reflect your production domain.
- Sitemap: `/sitemap.xml` (generated by `app/sitemap.ts`, static-only).
- Robots: `/robots.txt` (generated by `app/robots.ts`, references the sitemap).

## Deployment (Vercel)

1. Deploy the repo to Vercel.
2. Set env vars:
   - `SITE_URL` and `NEXT_PUBLIC_SITE_URL` to your production domain
   - `GITHUB_API_TOKEN` (optional, recommended)
   - `GROQ_API_KEY` (optional, enables AI recommendations)
